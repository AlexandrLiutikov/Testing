# AGENTS ENTRY POINT

Обязательное чтение перед началом любой задачи:

- docs/governance/RULES.md
- docs/governance/GIT_DISCIPLINE.md
- docs/governance/MASTER_PROMPT.md
- docs/governance/TASK_PROMPT.md

## Правила исполнения

- Без чтения указанных документов выполнение запрещено
- Для каждой новой задачи создавать новую рабочую ветку (main запрещен)
- Формат именования ветки task/<описание>
- Все изменения через PR
- Мержить самому запрещено
- Соблюдать docs/governance/RULES.md как источник истины

## Запрет на отклонения от правил

Агенту ЗАПРЕЩЕНО:
- придумывать архитектуру, не описанную в SCRIPT_RULES.md
- создавать файлы вне установленных слоёв (shared/, actions/, assertions/)
- писать код, не прочитав SCRIPT_RULES.md
- формировать RELEASE_DECISION без алгоритма из DECISION_ENGINE.md
- классифицировать FAIL без модели из MASTER_PROMPT.md (раздел 4)
- использовать любые решения, не зафиксированные в документации проекта

Если правило не описано явно — агент обязан спросить пользователя, а не принимать решение сам.

## Поведение

Если правила не загружены или нарушены — остановить выполнение задачи

## Предпочтения в работе
Для каждой задачи сначала укажите:
1. слой области видимости
2. канонические исходные файлы для обновления
3. файлы, которые следует оставить без изменений
4. команды проверки для выполнения

## Preflight (обязательный шаг)

Перед началом задачи агент обязан:
1. Прочитать все указанные документы
2. Вывести чек-лист подтверждения в формате:
[✓] docs/governance/RULES.md — прочитан
[✓] docs/governance/GIT_DISCIPLINE.md — прочитан
[✓] docs/governance/MASTER_PROMPT.md — прочитан
[✓] docs/governance/TASK_PROMPT.md — прочитан
Готов работать по правилам.

Если хотя бы один документ недоступен — остановить задачу и сообщить, какого файла не хватает.
