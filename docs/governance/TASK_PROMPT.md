STEP 1 — READ DOCUMENTATION (ОБЯЗАТЕЛЬНО)

Прочитай и проанализируй:

1. docs/governance/AGENTS.md

3. products/<Продукт>/<ПРОДУКТ>_RISK_MODEL.md

Если хотя бы один файл отсутствует:
→ остановись

──────────────────────────────────────

STEP 2 — EXTRACT CONSTRAINTS

Выведи:

RULES APPLIED:
- архитектурные ограничения (слои, reuse-first, запрет дублирования)
- правила написания скрипта
- требования к отчету
- требования к классификации сбоев

RISK MODEL IMPACT:
- какие части теста относятся к critical_path
- какие отклонения допустимы (tolerances)
- какие сбои приведут к NO_GO / GO_WITH_RISK

DECISION LOGIC:
- какие типы FAIL будут учитываться
- как будет формироваться RELEASE_DECISION

──────────────────────────────────────

STEP 3 — TASK MAPPING

Сопоставь задачу с правилами:

- какие semantic actions будут использоваться
- какие assertions подтвердят результат
- какие fallback допустимы (если понадобятся)
- какие шаги являются критическими

Выведи:

IMPLEMENTATION PLAN:
- структура сценария
- какие reusable функции будут использованы
- что нужно создать, если нет готового

──────────────────────────────────────

STEP 4 — VALIDATION

Перед началом реализации явно подтвердить:

- правила проекта учтены
- архитектура не нарушается
- дублирования не будет
- PASS будет ставиться только после assertion
- будет сформирован RELEASE_DECISION

Если есть сомнения или противоречия:
→ остановись и задай уточняющий вопрос

──────────────────────────────────────

STEP 5 — IMPLEMENTATION

Только после выполнения всех шагов выше:

- реализуй код строго по SCRIPT_RULES.md
- соблюдай разделение слоёв
- используй semantic actions / assertions
- не используй координаты без крайней необходимости
- не добавляй time.sleep() в test case
- не дублируй shared-логику

──────────────────────────────────────

STEP 6 — SELF-CHECK (ОБЯЗАТЕЛЬНО)

Перед завершением задачи выведи:

CHECK:
- какие правила применены
- есть ли нарушения (если есть — исправь)
- корректно ли классифицируются FAIL
- формируется ли RELEASE_DECISION
- соответствует ли отчет REPORT_STYLE_RULES.md

Если хоть один пункт не выполнен → исправь
