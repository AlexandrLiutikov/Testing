## AGENTS.md

### Новый раздел: «Правила при создании и модификации тестов»

| Правило | Суть |
|---------|------|
| Reuse-first | Перед написанием нового действия — обязательный поиск существующего в shared/actions/assertions |
| Правило второго использования | При втором появлении логики — немедленный вынос в reusable layer |
| Запрет копирования кода | Копирование между кейсами запрещено, если код может быть в shared |
| Предпочтительный порядок | 1) использовать готовое → 2) расширить reusable → 3) собрать сценарий |
| Приоритет поддерживаемости | Reusable-подход приоритетнее быстрого локального решения |

## Обязательные правила (сводка)

Следующие правила теперь являются **обязательными** и проверяемыми:

1. Платформенный код запрещён в Test Case Layer
2. Дублирование shared logic между кейсами запрещено
3. `time.sleep()` в Test Case Layer запрещён
4. PASS только после assertion (не внутри action)
5. Контракт step_result с обязательными полями
6. Классификация сбоев: TEST_FAIL / INFRA_FAIL / BLOCKED
7. Координатный fallback только с пометкой и причиной
8. Запрет обратных импортов (shared не импортирует из продукта)
9. Единые префиксы именования функций
10. Блок release_decision с полем run_confidence обязателен

## Архитектурные ограничения (сводка)

1. 5 обязательных слоёв с чёткими границами
2. Разделение shared/ (инфраструктура) и <Продукт>/actions/ (бизнес-логика)
3. Reuse-first: сначала ищем, потом пишем
4. Правило второго использования: дубликат → немедленный рефакторинг
5. Централизация wait/retry/timeout в Infrastructure Layer
6. Строгий порядок fallback: Accessibility → OCR → координаты → CV
